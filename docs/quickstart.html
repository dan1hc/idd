Create Repository from Template

Use GitHub CLI to scaffold your project

‚ñº

The fastest way to start is using IDD as a GitHub template. This gives you the complete framework with a single command.

bash 
gh repo create my-project --template dan1hc/idd
üí°

Don't have the GitHub CLI? Install it with brew install gh or download from cli.github.com

After creation, clone your new repo:

bash 
gh repo clone my-project && cd my-project

Bootstrap Your Codebase

Let AI map your existing code into features

‚ñº

The bootstrap process analyzes your codebase, identifies logical feature boundaries, and generates feature files with pre-populated glossaries.

bash 
.github/idd/compile.sh --bootstrap

Then run the loop (compile ‚Üí select agent.md ‚Üí prompt ‚Üí repeat) until complete.

‚ÑπÔ∏è

What happens: AI creates feature files like auth.md, billing.md based on your code structure, with glossaries already populated.

3

Review Generated Features

Check and refine the AI-generated feature files

‚ñº

After bootstrapping, review the generated feature files in .github/idd/features/. Each file should have:

Section Purpose
What Description of the feature
Acceptance Criteria Checklist of requirements
Glossary Semantic anchors to code locations

The glossary uses semantic anchors that survive refactoring:

markdown 
| What | Where |
|------|-------|
| Login handler | `src/auth.py::authenticate` |
| Validation logic | `src/auth.py::#user-auth:validate` |
| Auth tests | `tests/test_auth.py::TestLogin` |

Why IDD?

Without IDD With IDD
AI guesses your conventions AI detects and matches your patterns
No record of AI-generated code Glossary maps features ‚Üí code
Each session starts from zero Glossaries carry context forward
"Where did this code come from?" grep IDD:feature-name finds it
Refactoring breaks references Semantic anchors survive refactoring
